## 2024-03-XX - Replace `glob.glob` & `fnmatch` with `_glob_match` in `context_optimizer`
**Learning:** `glob.glob(recursive=True)` inside the hot path `_file_matches_pattern` is a significant performance bottleneck due to excessive filesystem scanning. Even with caching, it is inefficient. Pre-splitting the path strings on forward-slash matching strings using the discovery primitive `_glob_match` brings notable performance improvements because the paths inside compilation are generally all present inside `portable_relpath`.
**Action:** Use string matching / tokenization glob techniques (`_glob_match`) instead of the python builtin `glob.glob` when finding patterns against paths you are already walking or constructing manually. Always parse relative paths with `portable_relpath` first when testing paths globally.
